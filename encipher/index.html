Encipher v4.2

基於 AES-256-GCM (JWE) 標準的純客戶端本機加解密工具

🛠️ 工具用途

這是一個純客戶端 (Client-side) 的加密與解密工具，所有運算都在您的瀏覽器本機完成，資料絕不上傳，確保隱私安全。支援加密敏感文字或小型檔案，產生符合 JWE (JSON Web Encryption) 標準的密文。

⚠️ 版本相容性

此版本採用 JWE Compact Serialization 格式。與 v3.0 之前的版本不相容。在 JWE Protected Header 中新增了 app: "encipher" 與 v: 4 參數以識別密文來源。

⚙️ 技術規格 (v4.2)

  • 加密格式： JWE Compact Serialization (符合 RFC 7516 標準)。
  • 金鑰管理 (alg)：
    • dir (Direct Encryption) + Argon2id KDF (最高安全，預設)。
    • PBES2-HS512+A256KW (符合 RFC 7518 標準，最高相容)。
  • 對稱加密 (enc)： 固定採用 A256GCM (AES-256-GCM)。
  • 額外標頭： app: "encipher" 與 v: 4 (作為 AAD 驗證資料)。
💻 給開發者的話：實作相容的解密器

我們建議使用成熟的 JOSE / JWE 函式庫 (例如 Node.js 的 jose) 進行解密：

  1. 解析 JWE Protected Header 取得 alg。
  2. 若為 dir：讀取 kdf ("Argon2id"), salt, t, m, p 參數，並使用 Argon2id 衍生出 32-byte 金鑰，搭配 Header 作為 AAD 進行解密驗證。
  3. 若為 PBES2-HS512+A256KW：直接將密碼傳入 JWE 解密函式，函式庫會自動處理金鑰解包。
