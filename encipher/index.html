進階加密、解密器 v4

工具用途

這是一個純客戶端 (Client-side) 的加密與解密工具，所有運算都在您的瀏覽器本機完成，確保資料隱私。它可用於加密敏感文字或小型檔案，產生一個符合業界 JWE (JSON Web Encryption) 標準、可安全傳輸的密文。

版本 v4.0 說明

重要：此 v4.0 版本採用了業界標準的 JWE Compact Serialization 格式，旨在提供更好的互通性。它與先前的任何版本 (v1, v2, v3) 均不相容。

技術規格 (v4.0)

  • 加密格式： 最終輸出為一個 JWE Compact Serialization 字串。
  • 金鑰衍生 (KDF)： 支援 Argon2id (預設) 和 PBKDF2。產生的金鑰將直接用於加密 (alg: "dir")。
  • 對稱加密： 固定使用 A256GCM (AES-256-GCM) 作為加密演算法 (enc: "A256GCM")。

給開發者的話：實作相容的 v4.0 解密器

首選作法：
我們強烈建議您使用一個成熟的 JOSE / JWE 函式庫 (例如 Node.js 的 `jose`, Python 的 `jwcrypto` 等) 來處理解密。您只需要：

  1. 從 JWE Protected Header 中解析出 KDF 參數 (kdf, salt, t, m, p, i)。
  2. 使用這些參數與使用者密碼，重現出 32 位元組的金鑰。
  3. 將金鑰與完整的 JWE Compact 字串傳入函式庫的 `compactDecrypt` 方法即可。

手動實作的陷阱 (進階)：
若您必須手動實作，請務必注意 AAD (附加驗證資料) 的正確處理方式： 


// JWE Compact 格式的 AAD 是 Protected Header Base64URL 字串本身的 US-ASCII 位元組。
// 絕非對該字串進行 UTF-8 編碼後的結果！

const jweParts = jweString.split('.');
const protectedHeaderB64Url = jweParts[0];

// ❌ 錯誤的作法 (會導致認證失敗)
// const aad = new TextEncoder().encode(protectedHeaderB64Url);

// ✅ 正確的作法：直接將字串轉為 ASCII/Binary Bytes
const aad = new Uint8Array(protectedHeaderB64Url.length);
for (let i = 0; i < protectedHeaderB64Url.length; i++) {
    aad[i] = protectedHeaderB64Url.charCodeAt(i);
}

// 接著呼叫 crypto.subtle.decrypt(..., { ..., additionalData: aad }, ...)
